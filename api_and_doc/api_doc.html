Thanks! API Documentation

REST API for the Thanks community item-sharing platform. Allows givers to post free items and receivers to discover and claim them across Vietnam.

Base URLhttps://api.thanks.vn/v1
FormatJSON
AuthBearer JWT
Version1.0.0
🔑
Authentication

All protected endpoints require a Bearer token in the Authorization header:
Authorization: Bearer <token>

Tokens are returned by the /auth/otp/verify or /auth/social endpoints. Token lifetime is 30 days. Endpoints marked AUTH require a valid token.

🔓
Non-Logged In

These endpoints are publicly accessible. They power the unauthenticated browse experience and the sign-up / login flow (screens 2.1.5 and 2.1.6).

POST /auth/otp/send Send OTP to phone number
Screen 2.1.6 — Sign Up / Login

Sends a 6-digit one-time password via SMS to the given phone number. Works for both new sign-ups and returning users.

Request Body
FieldTypeRequiredDescription
phonestringrequiredVietnamese phone number, e.g. 0901234567
Example Request
{ "phone": "0901234567" }
Example Response — 200 OK
{ "message": "OTP sent", "expires_in": 300 }
POST /auth/otp/verify Verify OTP and receive JWT
Screen 2.1.6 — Sign Up / Login

Verifies the OTP entered by the user. On success, returns a JWT access token and the user object. Creates a new account if the phone number is new.

Request Body
FieldTypeRequiredDescription
phonestringrequiredSame phone number from send-OTP step
otpstringrequired6-digit code received via SMS
rolestringoptionalgiver or receiver — only needed on first sign-up
Example Response — 200 OK
{ "token": "eyJhbGci...", "user": { "id": "uuid-123", "phone": "0901234567", "name": null, "is_new_user": true } }
POST /auth/social Login with Zalo / Google / Facebook / Apple
Screen 2.1.6 — Sign Up

Authenticates the user using an ID token from a social provider. Returns a JWT on success. Creates a new account if the provider identity is new.

Request Body
FieldTypeRequiredDescription
providerstringrequired
zalogooglefacebookapple
id_tokenstringrequiredID token from the provider's SDK (not access_token)
namestringoptionalDisplay name hint from provider profile
avatar_urlstringoptionalAvatar URL hint from provider profile
emailstringoptionalEmail hint from provider profile
Dev mode: When the backend runs with JWT_SECRET=dev-only-secret-change-me, the id_token is decoded as a base64-encoded JSON object {"sub": "...", "email": "...", "name": "...", "picture": "..."} — no real provider SDK keys needed. Production provider verification is TODO per-provider.

Updates editable fields of an owned post. Cannot update if post has active claims (except pickup_window and description).

GET /messages/claims/:claim_id Get full message thread for a claim AUTH

Returns all messages in a claim's chat thread, ordered by created_at ascending. User must be either the giver or the receiver of the claim. Side effect: marks the peer's unread messages as read on each fetch.

For real-time messaging, connect to the WebSocket endpoint: wss://api.thanks.vn/ws/claims/:claim_id?token=<jwt>
Example Response — 200 OK
{ "claim_id": "uuid-claim", "peer": { "id": "uuid-giver", "name": "Bento Cooky", "avatar_url": "https://cdn.thanks.vn/...", "is_verified": true }, "messages": [ { "id": "uuid-msg", "sender_id": "uuid-receiver", "content": "Chào bạn, mình đến lúc 8h30 nhé", "is_read": true, "created_at": "2026-04-28T07:30:00Z" } ] }

POST /claims/:id/ratings Rate the other party after pickup AUTH
Post-pickup flow

Submits a star rating and optional comment for the other party in a completed claim. Givers rate receivers; receivers rate givers. Only allowed once per claim per user (UNIQUE constraint). Updates the rated user's rating_avg.

Request Body
FieldTypeRequiredDescription
scoreintrequired1–5 star rating
commentstringoptionalOptional written note (no max length currently enforced)
Errors
StatusError CodeWhen
409already_ratedCaller has already rated this claim (UNIQUE per claim + rater)
409claim_not_ratableClaim status is not picked_up or completed
403forbiddenCaller is not the giver or receiver of this claim
404claim_not_foundClaim ID does not exist
