Thanks! API Documentation

REST API for the Thanks community item-sharing platform. Allows givers to post free items and receivers to discover and claim them across Vietnam.

Base URLhttps://api.thanks.vn/v1
FormatJSON
AuthBearer JWT
Version1.0.0
🔑
Authentication

All protected endpoints require a Bearer token in the Authorization header:
Authorization: Bearer <token>

Tokens are returned by the /auth/otp/verify or /auth/social endpoints. Token lifetime is 30 days. Endpoints marked AUTH require a valid token.

🔓
Non-Logged In

These endpoints are publicly accessible. They power the unauthenticated browse experience and the sign-up / login flow (screens 2.1.5 and 2.1.6).

POST /auth/otp/send Send OTP to phone number
Screen 2.1.6 — Sign Up / Login

Sends a 6-digit one-time password via SMS to the given phone number. Works for both new sign-ups and returning users.

Request Body
FieldTypeRequiredDescription
phonestringrequiredVietnamese phone number, e.g. 0901234567
Example Request
{ "phone": "0901234567" }
Example Response — 200 OK
{ "message": "OTP sent", "expires_in": 300 }
POST /auth/otp/verify Verify OTP and receive JWT
Screen 2.1.6 — Sign Up / Login

Verifies the OTP entered by the user. On success, returns a JWT access token and the user object. Creates a new account if the phone number is new.

Request Body
FieldTypeRequiredDescription
phonestringrequiredSame phone number from send-OTP step
otpstringrequired6-digit code received via SMS
rolestringoptionalgiver or receiver — only needed on first sign-up
Example Response — 200 OK
{ "token": "eyJhbGci...", "user": { "id": "uuid-123", "phone": "0901234567", "name": null, "is_new_user": true } }
POST /auth/social Login with Zalo / Google / Facebook / Apple
Screen 2.1.6 — Sign Up

Authenticates the user using an OAuth access token from a social provider. Returns a JWT on success.

Request Body
FieldTypeRequiredDescription
providerstringrequired
zalogooglefacebookapple
access_tokenstringrequiredOAuth token from the provider's SDK
rolestringoptionalgiver or receiver — only for first sign-up
Example Response — 200 OK
{ "token": "eyJhbGci...", "user": { "id": "uuid-456", "name": "Nguyễn Văn A", "is_new_user": false } }

Returns the public profile of any user. Includes their recent/available/completed posts and stats. Private data (exact address) is hidden unless the viewer has an active claim from this user.

Giver

Endpoints for logged-in users acting as Givers — creating posts, managing their items, and tracking impact. All require AUTH.

POST /posts Create a new free item post AUTH
Screens 2.2.7 + 2.2.8 — Submit Item (Step 1 + Step 2)

Creates a new post. The request body covers both steps of the submit flow: Step 1 (photos, title, description) and Step 2 (category, quantity, pickup window, location, duration).

Request Body
FieldTypeRequiredDescription
titlestringrequiredItem name, max 200 chars
descriptionstringoptionalItem description (may be AI-generated on client)
categorystringrequired
foodclothesfurnituretechbooksother
image_urlsstring[]requiredOrdered list of uploaded image URLs (min 1). Upload files first via POST /uploads
quantityintrequiredTotal quantity available
quantity_per_receiverintoptionalMax a single receiver can claim (default: 1)
pickup_daysstring[]requirede.g. ["mon","tue","fri"]
pickup_start_timestringrequiredHH:MM format, e.g. "08:00"
pickup_end_timestringrequiredHH:MM format, e.g. "10:00"
latfloatrequiredPickup location latitude
lngfloatrequiredPickup location longitude
addressstringrequiredHuman-readable pickup address
duration_daysintrequiredPost lifespan in days (1 / 3 / 7 / 14 / 30)
business_iduuidoptionalIf posting on behalf of a business identity
Example Response — 201 Created
{ "id": "uuid-post-new", "status": "active", "closes_at": "2026-05-04T00:00:00Z" }
PUT /posts/:id Update an existing post AUTH

Updates editable fields of an owned post. Cannot update if post has active claims (except pickup_window and description).

DELETE /posts/:id Cancel and delete a post AUTH

Cancels an active post. All associated pending claims are automatically cancelled and claimants are notified.

POST /uploads Upload a file (photo/document) AUTH
Screens 2.2.7, 2.2.6, 2.3.4 — Anywhere with photo upload

Uploads a file to cloud storage. Returns a CDN URL. Use this before POST /posts, POST /businesses, or POST /organizations. Accepted: JPG, PNG, HEIC, WebP, PDF. Max 10 MB per file.

Request
Send as multipart/form-data with field name file.
Example Response — 200 OK
{ "url": "https://cdn.thanks.vn/uploads/abc123.jpg", "size_bytes": 204800 }

GET /messages List all conversations AUTH
Screens 2.1.3a, 2.1.3b — Messages (Receiver + Giver Views)

Returns a list of all message threads for the current user, grouped by claim. Each thread shows the last message preview and unread count. Response adapts based on whether the user is the giver or receiver in each claim.

GET /messages/claims/:claim_id Get full message thread for a claim AUTH

Returns all messages in a claim's chat thread, ordered by created_at ascending. User must be either the giver or the receiver of the claim.

For real-time messaging, connect to the WebSocket endpoint: wss://api.thanks.vn/ws/claims/:claim_id?token=<jwt>
POST /messages/claims/:claim_id Send a message in a claim thread AUTH
Screen 2.4.2 — Claim Confirmed (Message button)

Sends a message to the other party in a claim. Triggers a push notification to the recipient.

Request Body
FieldTypeRequiredDescription

POST /claims/:id/ratings Rate the other party after pickup AUTH
Post-pickup flow

Submits a star rating and optional comment for the other party in a completed claim. Givers rate receivers; receivers rate givers. Only allowed once per claim per user. Updates the rated user's rating_avg.

Request Body
FieldTypeRequiredDescription
scoreintrequired1–5 star rating
commentstringoptionalOptional written note, max 500 chars
POST /auth/logout Logout and invalidate token AUTH
Screen 2.1.4a/c — Profile (Sign out)

Invalidates the current JWT token server-side (adds it to a denylist). The client should also delete the token locally.

Example Response — 200 OK
{ "message": "logged out" }
